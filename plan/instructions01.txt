### Task 1.5 — Add missing database indexes

**Findings:** M-8, M-9, M-10, M-11, M-12, M-13, M-14

#### 1.5.1 Pre-index verification
For each table below, query existing indexes:
```sql

WHERE schemaname = 'mct_proddb' AND tablename = '<table>';
```

#### 1.5.2 Index inventory

| # | Migration name | Table | Index | Justification |
|---|---|---|---|---|
| 1 | `add_idx_attendances_employee_date` | `attendances` | `(employee_number, attendance_date)` | DTR generation in `routes/web.php:14-34, 36-55` |
| 2 | `add_idx_attendances_profile_date` | `attendances` | `(profile_id, attendance_date)` | DTR viewer per-profile lookups |
| 3 | `add_idx_leave_cards_profile_date` | `leave_cards` | `(profile_id, date_applied)` | `LeaveCardResource` table default sort + profile filter |
| 4 | `add_idx_service_records_profile_from` | `service_records` | `(profile_id, from)` | `LeaveApplicationResource.php:87-92` per-profile-select query |
| 5 | `add_idx_leave_inclusive_dates_application_date` | `leave_inclusive_dates` | `(leave_application_id, date)` | Duration sum joins |
| 6 | `add_idx_divisions_parent_sort` | `divisions` | `(parent_id, sort_order)` | Org tree load in `Pages/ListOrgStructure.php:31-40` |
| 7 | `add_idx_face_attendances_profile_date` | `face_attendances` | `(profile_id, attendance_date)` | Verify schema first; skip if already indexed |

#### 1.5.3 Production-safe index creation
PostgreSQL `CREATE INDEX` locks the table for writes. For production tables with significant row counts (especially `attendances`, `face_embeddings`), use `CREATE INDEX CONCURRENTLY`.

Laravel's `Blueprint::index()` does NOT emit `CONCURRENTLY`. You **MUST** use:
```php
public function up(): void {
    DB::statement('CREATE INDEX CONCURRENTLY IF NOT EXISTS idx_attendances_employee_date ON mct_proddb.attendances (employee_number, attendance_date)');
}

public function down(): void {
    DB::statement('DROP INDEX CONCURRENTLY IF EXISTS mct_proddb.idx_attendances_employee_date');
}
```

Important: `CREATE INDEX CONCURRENTLY` cannot run inside a transaction. You must disable Laravel's auto-transaction wrapping:
```php
public $withinTransaction = false;
```

Confirm this approach with `search-docs` query `"create index concurrently postgresql laravel migration"` before writing.

#### 1.5.4 Per-index protocol
For EACH index (one migration each):
1. Create migration: `php artisan make:migration add_idx_<table>_<cols> --no-interaction`.
2. Use the CONCURRENTLY pattern above.
3. `php artisan migrate --pretend` → post SQL.
4. Wait for written approval.
5. Run during a low-traffic window (ask human when).
6. Verify with `EXPLAIN ANALYZE` on the most-impacted query (e.g. `SELECT * FROM attendances WHERE employee_number = ? AND attendance_date = ?`). Confirm the new index is used.
7. Report `EXPLAIN` output.

#### 1.5.5 Rollback
Each migration has a `DROP INDEX CONCURRENTLY` `down()`. If a query plan regresses unexpectedly, roll back that one migration.

**Acceptance criteria:**
- All 7 indexes (minus any pre-existing) appear in `pg_indexes`.
- `EXPLAIN ANALYZE` on flagged queries shows the new index in use.
- Production query latency on impacted endpoints does not regress.

## 6. Emergency Procedures

### 6.1 If production breaks during Phase 1
1. **STOP all work immediately.**
2. Identify the offending commit via `git log` + correlation with deploy time.
3. Propose rollback via `git revert <sha>` (NOT `reset --hard`).
4. Post the proposed revert SQL/diff for human approval.
5. After approval, execute revert.
6. Investigate root cause; do not retry without a new plan and approval.

### 6.2 If a migration fails partway
1. Capture the error message verbatim.
2. Run `database-query` to inspect actual schema state vs expected.
3. Do NOT run additional migrations.
4. Report state to human; await instruction.
5. If a destructive change was made (column drop), check backup availability.

### 6.3 If Filament/Livewire components break
1. Run `php artisan optimize:clear` first — often a stale cache.
2. Run `php artisan filament:cache-components`.
3. Re-run `composer dump-autoload`.
4. If still broken, `git revert` the offending commit.
5. Report.
