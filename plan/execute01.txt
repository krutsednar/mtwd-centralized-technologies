## Inviolable rules (full list in instructions01.txt § 0; here are the absolute non-negotiables)

- **One task = one commit.** No mega-commits.
- **No destructive operations** without an explicit human directive in the conversation naming the operation. Forbidden list includes (non-exhaustive):
  - `git reset --hard`, `git push --force`, `git branch -D`, `git checkout -- .`, `git clean -fd`
  - `--no-verify` on any commit or push
  - `--force` on any migration command
  - `php artisan migrate:fresh`, `migrate:reset`, `db:wipe`, `db:seed` on production
  - `composer require`, `composer remove`, `composer update`
  - `npm install <pkg>`, `npm update`
  - Direct edits to `.env`, `composer.lock`, `package-lock.json`, `bootstrap/cache/*`, `vendor/*`, `storage/app/*`, `database/seeders/*`, `tests/Pest.php`, `tests/TestCase.php`
- **No skipping approval gates.** Phase 1 only. Phases 2 and 3 require separate written authorization.
- **No assuming Laravel/Filament/Livewire API behavior from training data.** Use Laravel Boost MCP (`search-docs`, `tinker`, `database-query`, `database-schema`, `list-artisan-commands`, `get-absolute-url`) to verify.
- **No merging your own PR.** Human reviews and merges.
- **Quote file paths with line numbers** when citing findings (e.g. `app/Models/Profile.php:113`).
- **Paste actual command output.** Do not paraphrase, do not summarise, do not abbreviate.
