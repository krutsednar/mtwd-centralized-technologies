================================================================================
 FACE BIOMETRICS ATTENDANCE — IMPLEMENTATION GUIDE (No CompreFace)
 Target: full capture + match in < 2 seconds
 Project: mtwd-centralized-technologies (Laravel 12 / Livewire / Alpine.js)
================================================================================

CURRENT STATE
-------------
Your app already has:
  - face-api.js loaded for face detection (TinyFaceDetector + FaceLandmark68Net)
  - Challenge-response liveness (head-turn verification)
  - AutoAttendance.php → calls CompreFace HTTP API for recognition
  - Biometrics.php    → calls CompreFace HTTP API for enrollment
  - Profile.face_enrolled boolean column

CompreFace is the only bottleneck. Every recognition makes an outbound HTTP
request to the CompreFace service (10-20 ms local, 200-800 ms if remote).
Replace it: do recognition entirely in the browser using face-api.js descriptors
stored in your own database.


WHY CLIENT-SIDE MATCHING IS FASTER
------------------------------------
  CompreFace flow:  capture → encode JPEG → HTTP POST → CompreFace process →
                    HTTP response → PHP validate → record  ≈ 300–1500 ms

  New flow:         capture → extract descriptor (browser) →
                    Euclidean match vs. preloaded descriptors →
                    HTTP POST only the employee_number → record  ≈ 80–250 ms

  The image never leaves the browser for recognition. Only a tiny JSON
  {employee_number, photo} hits the server after the match is confirmed.


================================================================================
 OPTION A — FULL CLIENT-SIDE (Recommended, simplest, fastest)
================================================================================

Uses face-api.js FaceRecognitionNet to produce a 128-dimension float descriptor
per face. Descriptors for all enrolled employees are loaded once on page open
(≈ 200 KB for 100 employees). Matching is a nearest-neighbour Euclidean search
done in <5 ms in the browser.

Accuracy: ~96–97% with a single descriptor per employee.
          ~98–99% if you store 3–5 descriptors and use the mean.

STEP 4 — Load employee descriptors on page init

Inside the Alpine.js component (attendanceClock or equivalent), add:

    // In the data() / init section:
    faceMatcher: null,

    async loadDescriptors() {
        const res = await fetch('/biometrics/descriptors');
        const employees = await res.json();

        const labeledDescriptors = employees.map(emp => {
            const descriptors = emp.descriptors.map(d => new Float32Array(d));
            return new faceapi.LabeledFaceDescriptors(
                emp.employee_number,   // used as the label for matching
                descriptors
            );
        });

        // 0.55 threshold = Euclidean distance cutoff (lower = stricter)
        // 0.5 is tight, 0.6 is lenient. Start at 0.55.
        this.faceMatcher = new faceapi.FaceMatcher(labeledDescriptors, 0.55);
        console.log(`Loaded ${labeledDescriptors.length} employee descriptors`);
    },

Call loadDescriptors() after model loading finishes:

    async captureAndMatch() {
        if (!this.faceMatcher) {
            this.status = 'Descriptors not loaded yet';
            return;
        }

        const detection = await faceapi
            .detectSingleFace(this.$refs.video, new faceapi.TinyFaceDetectorOptions())
            .withFaceLandmarks()
            .withFaceDescriptor();

        if (!detection) {
            this.status = 'No face detected';
            this.playFail();
            return;
        }

        const match = this.faceMatcher.findBestMatch(detection.descriptor);

        if (match.label === 'unknown') {
            this.status = 'Face not recognized';
            this.showFailModal = true;
            return;
        }

        // Optionally capture a photo for audit log
        const canvas = document.createElement('canvas');
        canvas.width = this.$refs.video.videoWidth;
        canvas.height = this.$refs.video.videoHeight;
        canvas.getContext('2d').drawImage(this.$refs.video, 0, 0);
        const photo = canvas.toDataURL('image/jpeg', 0.7);

        // Send only employee_number + photo to the server (no recognition needed server-side)
        @this.call('recordAttendance', match.label, photo);
    },

    async function enrollFace(employeeNumber) {
        // Load models first if not loaded
        await faceapi.nets.tinyFaceDetector.loadFromUri(MODEL_URL);
        await faceapi.nets.faceLandmark68Net.loadFromUri(MODEL_URL);
        await faceapi.nets.faceRecognitionNet.loadFromUri(MODEL_URL);

        const captures = [];
        const statusEl = document.getElementById('enroll-status');

        // Capture 5 descriptors from slightly different angles for robustness
        for (let i = 0; i < 5; i++) {
            statusEl.textContent = `Capturing ${i + 1}/5 — hold still...`;
            await new Promise(r => setTimeout(r, 600)); // small delay between captures

            const detection = await faceapi
                .detectSingleFace(videoEl, new faceapi.TinyFaceDetectorOptions())
                .withFaceLandmarks()
                .withFaceDescriptor();

            if (!detection) { i--; continue; }
            captures.push(Array.from(detection.descriptor)); // Float32Array → plain array for JSON
        }

        statusEl.textContent = 'Saving...';

        // Send to server
        await fetch('/biometrics/enroll', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'X-CSRF-TOKEN': document.querySelector('meta[name="csrf-token"]').content,
            },
            body: JSON.stringify({ employee_number: employeeNumber, descriptors: captures }),
        });

        statusEl.textContent = 'Enrolled!';
    }

Server-side enrollment handler (in Biometrics.php or a controller):

    public function enrollFace(Request $request): JsonResponse
    {
        $validated = $request->validate([
            'employee_number' => 'required|string|exists:profiles,employee_number',
            'descriptors'     => 'required|array|min:1|max:10',
            'descriptors.*'   => 'required|array|size:128',
        ]);

        $profile = Profile::where('employee_number', $validated['employee_number'])->firstOrFail();

        $profile->update([
            'face_descriptors' => $validated['descriptors'],
            'face_enrolled'    => true,
        ]);

        return response()->json(['status' => 'enrolled']);
    }

SECURITY HARDENING

1. SERVER-SIDE DESCRIPTOR VERIFICATION (optional, high-security mode)
   Send the descriptor array to the server along with employee_number.
   Server re-matches using a PHP Euclidean distance check:

    // In AutoAttendance.php
    public function recordAttendance(string $employeeNumber, string $photo, array $descriptor): void
    {
        $profile = Profile::where('employee_number', $employeeNumber)
            ->where('face_enrolled', true)
            ->firstOrFail();

        $storedDescriptors = $profile->face_descriptors; // array of 128-float arrays

        // Verify descriptor matches server-stored data
        $minDistance = PHP_FLOAT_MAX;
        foreach ($storedDescriptors as $stored) {
            $distance = $this->euclidean($descriptor, $stored);
            $minDistance = min($minDistance, $distance);
        }

        if ($minDistance > 0.55) {
            $this->showFailModal = true;
            return;
        }

        // ... proceed to record
    }

    private function euclidean(array $a, array $b): float
    {
        $sum = 0.0;
        for ($i = 0; $i < 128; $i++) {
            $diff = $a[$i] - $b[$i];
            $sum += $diff * $diff;
        }
        return sqrt($sum);
    }

------------------------------------------------------------------------
face_service.py  (save anywhere, e.g. C:\face-service\face_service.py)
------------------------------------------------------------------------

    from fastapi import FastAPI, File, UploadFile, HTTPException
    from fastapi.responses import JSONResponse
    import insightface
    import numpy as np
    import cv2, json, os

    app = FastAPI()

    # Load ArcFace model once at startup
    face_app = insightface.app.FaceAnalysis(
        name='buffalo_l',
        providers=['CUDAExecutionProvider', 'CPUExecutionProvider']
    )
    face_app.prepare(ctx_id=0, det_size=(320, 320))  # smaller det_size = faster

    EMBEDDINGS_FILE = 'embeddings.json'  # flat file cache; replace with DB calls

    def load_embeddings():
        if os.path.exists(EMBEDDINGS_FILE):
            with open(EMBEDDINGS_FILE) as f:
                return json.load(f)
        return {}

    def save_embeddings(data):
        with open(EMBEDDINGS_FILE, 'w') as f:
            json.dump(data, f)

    def cosine_similarity(a, b):
        a, b = np.array(a), np.array(b)
        return float(np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)))

    @app.post('/enroll')
    async def enroll(employee_number: str, file: UploadFile = File(...)):
        img_bytes = await file.read()
        img = cv2.imdecode(np.frombuffer(img_bytes, np.uint8), cv2.IMREAD_COLOR)
        faces = face_app.get(img)
        if not faces:
            raise HTTPException(400, 'No face detected')
        embedding = faces[0].normed_embedding.tolist()
        db = load_embeddings()
        if employee_number not in db:
            db[employee_number] = []
        db[employee_number].append(embedding)
        save_embeddings(db)
        return {'status': 'enrolled', 'employee_number': employee_number}

    @app.post('/recognize')
    async def recognize(file: UploadFile = File(...)):
        img_bytes = await file.read()
        img = cv2.imdecode(np.frombuffer(img_bytes, np.uint8), cv2.IMREAD_COLOR)
        faces = face_app.get(img)
        if not faces:
            return JSONResponse({'match': None, 'reason': 'no_face'})

        query_emb = faces[0].normed_embedding
        db = load_embeddings()

        best_label, best_score = None, -1.0
        for emp_number, embeddings in db.items():
            for emb in embeddings:
                score = cosine_similarity(query_emb, emb)
                if score > best_score:
                    best_score = score
                    best_label = emp_number

        THRESHOLD = 0.40  # cosine similarity; tune empirically (0.35–0.45)
        if best_score < THRESHOLD:
            return JSONResponse({'match': None, 'score': best_score, 'reason': 'below_threshold'})

        return JSONResponse({'match': best_label, 'score': best_score})

    if __name__ == '__main__':
        import uvicorn
        uvicorn.run(app, host='127.0.0.1', port=7860)

================================================================================
 QUICK THRESHOLD TUNING GUIDE
================================================================================

  face-api.js uses EUCLIDEAN DISTANCE (0 = identical, 1+ = very different)
  InsightFace uses COSINE SIMILARITY  (1 = identical, 0 = unrelated)

  face-api.js FaceMatcher threshold (distance):
    0.45 — very strict, may reject valid users in different lighting
    0.55 — recommended starting point
    0.60 — lenient, small chance of false positives with look-alikes
    0.65 — too lenient for production

  InsightFace cosine threshold:
    0.35 — strict
    0.40 — recommended
    0.45 — lenient

  To tune: enroll 3 people, test recognition under:
    - Different lighting (bright room vs dim)
    - Glasses on/off
    - Different angles (±20 degrees)
  Adjust threshold until false-reject rate < 2% and false-accept = 0.
